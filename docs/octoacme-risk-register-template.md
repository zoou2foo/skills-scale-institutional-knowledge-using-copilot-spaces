## Risk Scoring Guide

Use the tables below to assign **Impact** and **Probability** scores to each risk, then calculate the **Combined Score** (Impact × Probability) to prioritize mitigation efforts.

### Impact
| Score | Level | Description |
|---|---|---|
| 1 | Low | Minimal effect on schedule, cost, or quality |
| 2 | Medium | Noticeable delay or rework required |
| 3 | High | Significant impact on delivery, budget, or user experience |
| 4 | Critical | Project failure, major compliance breach, or significant customer harm |

### Probability
| Score | Level | Description |
|---|---|---|
| 1 | Unlikely | Rare; would require unusual circumstances |
| 2 | Possible | Could occur under foreseeable conditions |
| 3 | Likely | Expected to occur without active mitigation |
| 4 | Near Certain | Will almost certainly occur if not addressed immediately |

### Combined Score Thresholds
| Score Range | Priority | Action Required |
|---|---|---|
| 1–3 | LOW | Monitor; review at next planning cycle |
| 4–6 | MEDIUM | Assign owner; define mitigation plan |
| 8–12 | HIGH | Escalate to project sponsor; active mitigation underway |
| 16 | CRITICAL | Immediate escalation and executive attention required |

## Risk Categories Reference

| Category | Examples |
|---|---|
| **Schedule** | Dependencies delayed, key personnel unavailable, scope creep |
| **Technical** | Integration failures, performance issues, unknown third-party behavior |
| **Resource** | Budget cuts, team turnover, tooling unavailability |
| **Compliance** | Regulatory changes, audit findings, contractual obligation failures |
| **Security** | Vulnerability discovered in dependencies, unauthorized access, data breach |
| **External** | Vendor delays, market changes, infrastructure outages |
